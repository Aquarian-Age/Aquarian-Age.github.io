rust-ca

官方脚本

curl https://sh.rustup.rs -sSf | sh

  • 下载报错 从源安装倒是可以然而编译的时候下载其他包依然报同样的错误　看提示是证书问题　google了Ｎ多个地方说什么的都有　然而问题依旧

错误提示

info: syncing channel updates for ‘stable-x86_64-unknown-linux-gnu’
error: could not download file from ‘https://static.rust-lang.org/dist/channel-rust-stable.toml.sha256‘ to ‘/home/xuan/.rustup/tmp/zck1mpa6f2old3k6_file

下载脚本到本地

curl -o file https://sh.rustup.rs

手动下载证书

curl -o ca https://static.rust-lang.org/dist/channel-rust-stable.toml.sha256

  • 这意思我的ＤＮＳ解析有问题吗? 现有的dns规则已经改为手动了看了一下resolv.conf里面的搜索域还是很早时候的
    改过来和hosts一样的

hostname liangzi.local

/etc/resolve.conf search SUSE.suse改为 seach liangzi.local

systemctl restart nscd.service

curl -o ca https://static.rust-lang.org/dist/channel-rust-stable.toml.sha256

  • 果然是dns解析这里　到这里手动下载证书是没问题了　然而运行脚本还是歇菜

cat ca
e611—-9c0 channel-rust-stable.toml

错误提示2

info: caused by: https://static.rust-lang.org/dist/channel-rust-stable.toml.sha256: error trying to connect: error:1416F086:SSL routines:tls_process_server_certificate:certificate verify failed:ssl/statem/statem_clnt.c:1924: (unable to get local issuer certificate)

  • 这意思是网络原因了？挂代理都不行　这指定不是网络原因了
  • ls /etc/ssl里面都没这个statem
  • 搜了一下几个ca的包发现了三个中有两个没有装　ca-certificates-steamtricks　这个装完在运行脚本搞定了….
  • ls /etc/ssl/certs 有了
  • 期间还怀疑是脚本问题　定位到这个错误出现的原因是在判断系统的时候有个armv7注释掉整if语句还是不行　虽然在man函数中正常返回了系统类型
